# Plans

This directory contains plan documents for significant work in `halos-spec`.

Each plan describes **what** is being done, **why**, and **how** — with enough detail that an agent or contributor can pick it up and execute. Plans are living documents updated as work progresses.

## Conventions

- One plan per file: `NNN-short-name.md` (e.g., `001-repo-reorganization.md`)
- Each plan links to its tracking issue(s) on GitHub
- Plans move through states: `draft` → `active` → `complete`
- Plans that are superseded or abandoned are marked as such, not deleted

## Relationship to Issues

Plans describe the *design and rationale*. Issues track the *work items*. A plan may spawn multiple issues. An issue references its parent plan.
